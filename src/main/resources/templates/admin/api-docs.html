🏟️ Cheerlot Server API 명세서

📋 프로젝트 개요

🎯 목적

KBO(한국야구위원회) 라인업 및 응원가 데이터를 제공하는 REST API 서버

🏗️ 주요 기능

  • KBO 선수 및 팀 정보 관리
  • 실시간 라인업 데이터 크롤링
  • 선수별 응원가 오디오 파일 제공
  • 버전 관리 시스템
  • 웹 기반 관리자 인터페이스

🌍 Base URL

http://15.164.33.36:8080

📖 API 엔드포인트 명세

👥 선수 관리 (Player API)

전체 선수 조회

GET /api/players

설명: 모든 KBO 선수 정보를 조회합니다.

응답 예시:
[
    {
        "playerId": "LG:10",
        "name": "이민호",
        "backNumber": "10",
        "position": "외야수",
        "batsThrows": "우타 좌투",
        "batsOrder": "1",
        "teamCode": "LG",
        "cheerSongs": [
            {
                "id": "lg10-cheer-1",
                "title": "이민호 응원가",
                "lyrics": "이민호 이민호 우리의 스타\\n홈런 홈런 날려라",
                "audioFileName": "lg10.mp3",
                "playerId": "LG:10"
            }
        ]
    }
]
응답 코드:

200 OK 성공적으로 조회됨

팀별 선수 조회

GET /api/players/{teamCode}

설명: 특정 팀의 선수 정보를 타순 순으로 조회합니다.

파라미터:
teamCode string 필수 팀 코드 (LG, KIA, SSG, LT, HH, NC, OB, SK, HT, WO)
응답 코드:

200 OK 성공적으로 조회됨
404 Not Found 해당 팀을 찾을 수 없음

📋 라인업 관리 (Lineup API)

팀별 라인업 조회

GET /api/lineups/{teamCode}

설명: 특정 팀의 현재 라인업 정보를 조회합니다. (타순이 있는 선수만 포함)

응답 예시:
{
    "updated": "09월 07일",
    "opponent": "vs KIA",
    "players": [
        {
            "playerId": "LG:10",
            "name": "이민호",
            "backNumber": "10",
            "position": "외야수",
            "batsThrows": "우타 좌투",
            "batsOrder": "1",
            "teamCode": "LG",
            "cheerSongs": [...]
        }
    ]
}

수동 크롤링 실행

GET /api/crawl

설명: 경기 일정 및 라인업 데이터를 수동으로 크롤링합니다.

응답 예시:
"크롤링 완료"

🎵 응원가 관리 (CheerSong API)

응원가 오디오 파일 조회

GET /api/cheersongs/{code}

설명: 선수별 응원가 오디오 파일을 제공합니다.

파라미터:
code string 필수 응원가 파일 코드 (팀코드 + 백넘버 + .mp3/.mp4)
예시 요청:
GET /api/cheersongs/lg10.mp3
GET /api/cheersongs/kia7.mp3
GET /api/cheersongs/ssg39.mp3
응답:

Content-Type: audio/mpeg (mp3) 또는 audio/mp4 (mp4)
Binary audio data (음성 파일 스트림)

📊 버전 관리 (Version API)

로스터 버전 조회

GET /api/version/roster/{teamCode}/number

설명: 특정 팀의 로스터 버전 번호를 조회합니다. 로스터 정보가 변경될 때마다 버전 번호가 1씩 증가합니다.

응답 예시:
12

라인업 버전 조회

GET /api/version/lineup/{teamCode}/number

설명: 특정 팀의 라인업 버전 번호를 조회합니다. 라인업 정보가 변경될 때마다 버전 번호가 1씩 증가합니다.

응답 예시:
8

🔧 관리자 API (Admin API)

관리자 API는 HTML 페이지를 반환하는 웹 인터페이스입니다.
주요 엔드포인트:
  • GET /admin - 관리자 대시보드
  • GET /admin/players - 선수 관리 페이지
  • GET /admin/lineup - 라인업 관리 페이지
  • GET /admin/cheersongs - 응원가 관리 페이지
  • POST /admin/player/{id}/batting-order - 선수 타순 수정

🏟️ 팀 코드 정보

  • LG - LG 트윈스 (LG Twins)
  • KIA - KIA 타이거즈 (KIA Tigers)
  • SSG - SSG 랜더스 (SSG Landers)
  • LT - 롯데 자이언츠 (Lotte Giants)
  • HH - 한화 이글스 (Hanwha Eagles)
  • NC - NC 다이노스 (NC Dinos)
  • OB - 두산 베어스 (Doosan Bears)
  • SK - SK 와이번스 (SK Wyverns)
  • HT - KT 위즈 (KT Wiz)
  • WO - 키움 히어로즈 (Kiwoom Heroes)

📝 참고사항

  • 모든 API는 JSON 형태로 응답합니다
  • 응원가 파일은 /cheersongs/audio/ 디렉터리에 저장됩니다
  • 자동 크롤링은 스케줄링되어 주기적으로 실행됩니다
  • 관리자 인터페이스는 Thymeleaf 템플릿을 사용합니다
  • 타순이 "0"인 선수는 라인업에서 제외됩니다
  • 응원가 파일 형식: {팀코드소문자}{백넘버}.mp3 또는 .mp4 (예: lg10.mp3, kia7.mp3)
  • 모든 날짜는 한국 시간(KST) 기준입니다
